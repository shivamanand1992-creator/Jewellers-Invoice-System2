# Complete System Overview: S.S. Jewellers Invoicing System v2

Let me explain everything we've built in detail:

---

## 🎯 **What This System Does**

This is a **professional invoicing application for a jewellery shop** that helps create, manage, and download professional invoices as PDFs. It's built for S.S. Jewellers in Delhi and handles:

- **Multi-item invoices** (one invoice can have gold rings, silver bracelets, etc.)
- **Per-item making charges** (each item can have a different making charge percentage)
- **Automatic GST calculations** (3% on jewellery, 5% on making charges)
- **Professional PDF generation** (with QR code, watermark, shop details)
- **Invoice tracking** (see all past invoices, total sales, total GST collected)

---

## 🏗️ **Architecture: Frontend + Backend + Database**

```
User's Browser
    ↓
React Frontend (Web Interface)
    ↓
Express Backend (Node.js Server)
    ↓
PostgreSQL Database (Stores invoices, users, items)
```

### **Frontend (What You See)**
- Built with **React 18** 
- 5 pages: Login, Register, Dashboard, Create Invoice, View Invoice
- Mobile-responsive design (works on phones, tablets, laptops)
- Uses **Webpack** to bundle code for browsers

### **Backend (The Brain)**
- Built with **Node.js + Express**
- Handles all business logic
- Generates PDFs with **pdfkit**
- Authenticates users with **JWT tokens**
- Connects to PostgreSQL database

### **Database (The Memory)**
- **PostgreSQL** stores everything
- 3 main tables: `users`, `invoices`, `invoice_items`
- Auto-initializes on first startup

## 💻 **User Flow: How It All Works Together**

### **Step 1: User Logs In**
```
User enters email & password
         ↓
Browser sends to: POST /api/login
         ↓
Server checks password against database
         ↓
Server creates JWT token (signed with secret key)
         ↓
Token sent back to browser, stored in localStorage
         ↓
User sees Dashboard
```

**What's a JWT token?**
- A special string that proves the user is logged in
- Contains encoded user ID
- Expires after 7 days
- Sent with every API request in the Authorization header

---

### **Step 2: User Creates Invoice**

**A. Frontend (React)**
```
User fills form:
  - Customer name ← REQUIRED
  - Customer address ← optional
  - Customer GSTIN ← optional
  - Items (gold ring, silver bracelet, etc.)

For each item:
  - Description ← REQUIRED
  - Gross weight ← optional
  - Net weight ← REQUIRED for weight-based items
  - Price per gram ← how much per gram of gold/silver
  - Gemstone price ← extra stone cost
  - Making charge % ← e.g., 10% for this item

User clicks "Preview & Submit"
         ↓
Frontend calculates totals for each item
         ↓
Shows preview modal with all items and calculations
         ↓
User clicks "Create Invoice"
```

**B. Backend (Node.js + Express)**
```
Receives POST /api/invoices with all the data
         ↓
Verifies token (checks user is logged in) ✓
         ↓
Gets next invoice number (INV-001, INV-002, etc.)
         ↓
Calculates total:
  For each item:
    - Amount = (net_weight × price_per_gram) + gemstone_price
    - Making charge = amount × making_charge_%  / 100
    - GST on jewellery = amount × 3% = 525 (example)
    - GST on making = making_charge × 5% = 78.75 (example)
    - Item total = amount + making + gst_jewel + gst_making
  
  Grand total = sum of all items
         ↓
Inserts invoice into database
         ↓
Inserts each item into database
         ↓
Returns invoice_number: "INV-001" to frontend
         ↓
Frontend redirects to Dashboard
         ↓
Dashboard auto-refreshes and shows new invoice
```

## 🧮 **Math Behind the Scenes**

### **Example: Gold Ring with Gemstone**

Input from user:
```
Item Type: Gold
Description: Gold Ring with Diamond
Net Weight: 2.5g
Price per gram: ₹7000
Gemstone Price: ₹5000
Making Charge: 10%
```

Calculations:
```
1. Item Amount
   = (Net Weight × Price/gram) + Gemstone Price
   = (2.5 × 7000) + 5000
   = 17500 + 5000
   = ₹22500

2. Making Charge
   = Item Amount × Making Charge % / 100
   = 22500 × 10 / 100
   = ₹2250

3. GST on Jewellery (3%)
   = Item Amount × 3 / 100
   = 22500 × 3 / 100
   = ₹675

4. GST on Making (5%)
   = Making Charge × 5 / 100
   = 2250 × 5 / 100
   = ₹112.50

5. Item Total
   = Amount + Making + GST_Jewel + GST_Making
   = 22500 + 2250 + 675 + 112.50
   = ₹25537.50
```

## 🔐 **Security: JWT Authentication**

### **How it works:**

**Login:**
```
User: "email: shiv@gmail.com, password: Today@123"
         ↓
Server hashes password and compares
         ↓
If match, creates token:
  payload = { userId: 1, iat: 1778781992 }
  token = JWT.sign(payload, SECRET_KEY)
  Returns: "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
         ↓
Browser stores in localStorage
```

**Using the token:**
```
User clicks "View Invoices"
         ↓
Frontend sends request:
  GET /api/invoices
  Header: Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
         ↓
Server receives request
         ↓
Middleware extracts token from header
         ↓
Verifies token signature (checks it wasn't tampered with)
         ↓
Extracts userId = 1
         ↓
Queries: SELECT * FROM invoices WHERE user_id = 1
         ↓
Returns only THIS user's invoices
         ↓
User sees their invoices only
```

**Why this is secure:**
- Password is hashed (never stored in plain text)
- Token is signed (can't be forged)
- Token expires (stolen tokens become useless after 7 days)
- Each request is verified (server checks token is valid)

## 🚀 **Deployment: How It Goes Live**

### **Local → GitHub → Railway**

```
1. LOCAL DEVELOPMENT
   - Code on your computer
   - npm start to run locally
   - Test everything

2. GIT COMMIT
   - git add -A
   - git commit -m "message"
   - Code saved with message

3. GITHUB PUSH
   - git push origin main
   - Code uploaded to GitHub
   - Visible at github.com/shivamanand1992-creator/jewellery-invoicing-v2

4. RAILWAY DEPLOYMENT
   - Railway watches GitHub repo
   - When code is pushed, Railway:
     a. Pulls code from GitHub
     b. Runs npm install (installs dependencies)
     c. Runs npm run build (compiles React)
     d. Runs npm start (starts server)
     e. Creates PostgreSQL database
     e. Assigns public URL (https://...)
   - Your app is now LIVE on the internet

5. LIVE USAGE
   - Users go to Railway URL
   - Login with email/password
   - Create invoices
   - Download PDFs
   - Everything is stored in PostgreSQL
```

- First invoice: INV-001

### **5. Dashboard with Stats**
- Total invoices created

- Monthly GST breakdown

### **6. Mobile Responsive**
